Padb: A parallel debugging tool

About

Padb is a tool for examining and debugging parallel programs, originally developed at Quadrics it's now maintained externally and supports a wide range of parallel environments. It's an open source, non-interactive, command line, scriptable tool intended for use by programmers and system administrators alike.

Features

  • Stack trace generation
  • MPI Message queue display
  • Deadlock detection
  • Process interrogation
  • Signal forwarding
  • MPI collective reporting

What padb isn't

Padb is a job inspection tool, it can tell you want you want to know about your job and your MPI stack, it however will not tell you about your cluster as a whole and it won't diagnose problems with your wider environment, including you job launcher. Padb does not and will not launch or wrap your jobs for you, it is not a job harness.

Licence

padb is licenced under the LGPL and as such is open-source.

Prerequisites

Padb requires very little support from the OS or parallel environment to run, it's main use is to assist in the debugging of parallel applications, it's assumed that you have a working MPI stack and that "Hello world" application runs to completion without error.
A Linux operating system is assumed and a working gdb is required for stack trace functionality.
At the time of writing a shared filesystem between all nodes in the parallel job is required if using orte (Open-MPI runtime).

Parallel Environments

Padb works and is supported on the following parallel environments and MPI stacks. Not all features are available on all runtimes.
  • Quadrics RMS
    • Runs natively on clusters running RMS.
  • Slurm
    • Runs natively on clusters running Slurm irrespective of the runtime used.
  • Open MPI
    • Supports orted or OMPI jobs run under slurm.
  • MPICH2
    • Supports mpd or MPICH2 jobs run under slurm.

Mailing Lists

Mailing lists for padb discussion and development are available for public use.
  • padb-devel archives.
  • padb-users archives.

Patches

The advanced Group Deadlock Detection algorithm within padb requires modifications to the MPI library to function properly. At this time patches are available for Open MPI v1.3 and svn trunk only, contact padb-devel@pittman.org.uk for updates.

Building

Padb is written in perl and can be run immediately after downloading. Some functionality however requires binaries to be built on the target system. To build these download the tarball and run make followed by make install.
